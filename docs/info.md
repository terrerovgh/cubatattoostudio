Project Blueprint & AI Development Protocol: Cubatattoo StudioParte I: Fundamento Estratégico y ArquitectónicoEsta parte inicial establece el "porqué" y el "cómo" del proyecto. Proporciona al agente de IA el contexto fundamental necesario para tomar decisiones inteligentes y alineadas a lo largo de todo el proceso de desarrollo.Sección 1: Carta del Proyecto e Identidad de Marca1.1. Visión y Objetivos del ProyectoEl objetivo principal es crear un buque insignia digital para Cubatattoo Studio. Este sitio web no es simplemente un portafolio, sino el centro neurálgico para la adquisición de clientes, la comunicación de la marca y el establecimiento de la autoridad artística del estudio. El éxito del proyecto se medirá por su capacidad para convertir visitantes en clientes con citas agendadas, al tiempo que se refuerza la reputación del estudio como un referente de alta calidad en el arte del tatuaje.El sitio web debe fusionar a la perfección una estética de alto nivel artístico con una funcionalidad clara y centrada en el usuario. Debe servir como la fuente definitiva de información sobre los artistas, las políticas del estudio (incluyendo higiene, procesos de reserva y depósitos), y consejos de cuidado posterior al tatuaje.1 Al centralizar esta información, el sitio web no solo generará confianza en los clientes potenciales, sino que también reducirá la carga administrativa del personal del estudio al responder proactivamente a las preguntas más frecuentes.1.2. Estética de Marca: "Modern Grit" (Determinación Moderna)La identidad visual del proyecto se define como "Modern Grit". Este concepto combina la energía cruda y vanguardista de la cultura clásica del tatuaje con la presentación limpia y sofisticada del diseño digital moderno. El resultado debe ser una experiencia visual que se sienta a la vez auténtica y profesional. La implementación de esta estética se logrará mediante el uso de fotografía dramática de alta resolución, presentada sobre un diseño minimalista que utilice generosamente el espacio en blanco para permitir que las imágenes respiren y hablen por sí mismas.3Paleta de Colores: La paleta se ha seleccionado para evocar sofisticación y audacia.Primario (Fondo): Casi Negro (#111111). Proporciona un fondo dramático que hace que las imágenes y el texto resalten.Primario (Texto): Blanco (#FFFFFF). Garantiza la máxima legibilidad sobre el fondo oscuro.Acento: Rojo Audaz (#B91C1C). Este color se reservará para llamadas a la acción (CTAs), enlaces importantes y elementos destacados. Su uso se inspira en ejemplos de sitios de tatuajes exitosos que utilizan un color de acento fuerte para guiar al usuario, como el caso de "Nokomis Tattoo".3Secundario: Gris (#A1A1AA). Se utilizará para texto menos prominente, bordes sutiles y metadatos, proporcionando una jerarquía visual sin competir con los colores primarios.Tipografía: La elección tipográfica es fundamental para comunicar la dualidad de la marca: fuerza y claridad.Encabezados: Se utilizará una fuente sans-serif audaz y ligeramente condensada (por ejemplo, Anton u Oswald). Este tipo de letra transmite fuerza y confianza, alineándose con la naturaleza audaz del arte del tatuaje.5Cuerpo de Texto: Se empleará una fuente sans-serif limpia y altamente legible (por ejemplo, Inter o Roboto). La claridad es primordial para la información funcional como biografías de artistas, políticas y FAQs.51.3. Perfil del Público Objetivo: "El Coleccionista Exigente"El usuario objetivo de este sitio web no es el cliente impulsivo que busca un tatuaje por primera vez. Es un "coleccionista" de entre 25 y 45 años que considera los tatuajes como una forma de arte de alto valor. Este individuo investiga a los artistas de forma exhaustiva, valora un estilo único y un alto nivel de profesionalismo, y está dispuesto a invertir significativamente en una obra de calidad.Este perfil de usuario es digitalmente competente, activo en plataformas visuales como Instagram, y espera una experiencia en línea impecable y profesional desde la investigación inicial hasta la reserva y el cuidado posterior.6 El sitio web debe, por lo tanto, apelar a su apreciación por el arte y, al mismo tiempo, satisfacer su expectativa de un servicio profesional y eficiente.La estructura y el contenido del sitio web deben reflejar un entendimiento profundo de este público. Los análisis de sitios de tatuajes de primer nivel revelan un patrón consistente: los más exitosos logran un equilibrio delicado entre funcionar como una galería de arte y como un proveedor de servicios profesionales.2 Un sitio que se inclina demasiado hacia lo artístico puede resultar confuso o poco funcional, mientras que uno excesivamente corporativo puede no lograr conectar emocionalmente con un público que busca una expresión artística única. La definición explícita de la estética "Modern Grit" y el perfil del "Coleccionista Exigente" proporciona al agente de IA el contexto necesario para navegar este equilibrio, creando un diseño que sirve tanto a la fase de "inspiración" (navegar por el arte) como a la de "acción" (reservar una cita).Sección 2: Arquitectura Técnica y Configuración del Entorno2.1. Se elige Next.js por sus capacidades de Renderizado del Lado del Servidor (SSR) y Generación de Sitios Estáticos (SSG). Estas características son cruciales para un rendimiento superior en la optimización para motores de búsqueda (SEO) y tiempos de carga rápidos, factores determinantes para retener el interés del usuario en un sitio web visualmente intensivo.2Lenguaje: TypeScript. Se utilizará TypeScript para garantizar la seguridad de tipos en toda la aplicación. Esto es fundamental para construir una base de código escalable y mantenible, ya que reduce drásticamente la probabilidad de errores en tiempo de ejecución. La biblioteca reactbits.dev proporciona fragmentos de código tanto en JavaScript como en TypeScript, lo que convierte a TypeScript en una elección natural y preferida.9Estilos: Tailwind CSS. Se adopta este framework de CSS "utility-first" para permitir un desarrollo rápido y consistente de la interfaz de usuario directamente en el código JSX. La compatibilidad explícita de reactbits.dev con Tailwind CSS simplifica enormemente la integración de sus componentes, asegurando que el estilo se pueda aplicar y modificar de manera eficiente.92.2. La naturaleza no convencional de reactbits.dev, que se basa en copiar y pegar código en lugar de una instalación a través de npm, introduce un riesgo de mantenimiento futuro.9 Sin una estrategia arquitectónica clara desde el principio, sería imposible distinguir entre los componentes personalizados y los "prestados", lo que complicaría enormemente las actualizaciones o la corrección de errores. Protocolo de Integración de Componentes (AIOp - Procedimiento Operativo de IA)Existe una brecha crítica entre la comprensión implícita de un desarrollador humano y la necesidad de instrucciones explícitas de una IA. Un humano sabe buscar dependencias en la documentación; una IA podría no hacerlo. El flujo de trabajo no estándar de reactbits.dev exacerba esta brecha. Por lo tanto, el agente de IA debe seguir este protocolo preciso para cada componente de reactbits.dev que se le indique utilizar:Navegar: Acceder mediante programación a la URL proporcionada para el componente especificado en reactbits.dev.Configurar: En el visor de código de la página, seleccionar las pestañas "TypeScript" y "Tailwind".Verificar Dependencias: Leer la sección "Install dependencies" en la página del componente.12 Si se listan nuevas dependencias que no han sido instaladas previamente, ejecutar el comando npm install [nombre-de-la-dependencia] en la terminal.Copiar Código: Copiar todo el contenido del código para el componente.Crear Archivo: Crear un nuevo archivo dentro del directorio /src/components/react-bits/. El nombre del archivo debe ser exactamente el nombre del componente en formato PascalCase (ej. Masonry.tsx).Pegar y Guardar: Pegar el código copiado en el nuevo archivo y guardarlo.Importar y Usar: Importar el componente desde su nueva ruta local (ej. import Masonry from '@/components/react-bits/Masonry';) en la página o componente de destino.4.2. **Directivas Fundamentales:**
1.  **Adhesión al Blueprint:** Debe seguir la Arquitectura Técnica, la Estructura de Directorios y el Mapeo de Componentes definidos en el Blueprint.
2.  **Protocolo de Reactbits.dev:** Cuando se le instruya usar un componente de reactbits.dev, DEBE seguir el "Protocolo de Integración de Componentes (AIOp)" detallado en la Sección 4.1 del Blueprint. Esto incluye verificar e instalar dependencias ANTES de usar el componente.
3.  **Implementación por Fases:** Trabajará de manera secuencial y por fases. Construirá una página o un componente principal a la vez, presentará los cambios de código para su revisión y esperará la aprobación del usuario antes de pasar a la siguiente tarea.
4.  **Calidad del Código:** Todo el código generado debe ser limpio, bien comentado y adherirse a las mejores prácticas de TypeScript y Tailwind CSS.
5.  **Comunicación:** Comunicará su plan antes de ejecutarlo. Por ejemplo: "Ahora crearé la página de Portafolio. Instalaré `react-masonry-css` y luego implementaré el componente `Masonry` de reactbits.dev como se especifica en la Tabla 4.1".
