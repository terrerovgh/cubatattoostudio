**Cómo Usar esta Tabla para tu GitHub Project:**

1.  **Revisa las Tareas del MVP:** Confirma que estas son las tareas en las que te quieres enfocar para la próxima semana. Ajusta según sea necesario.
2.  **Crea Issues en GitHub:** Para cada tarea de la sección "MVP - Foco Principal", crea un nuevo "Issue" en tu repositorio `cubatattoostudio`.
    * Usa la "Tarea / Descripción del Issue" como título del Issue.
    * En la descripción del Issue, puedes añadir más detalles si quieres.
    * **Aplica Labels (Etiquetas):**
        * Crea y aplica etiquetas para el "Esfuerzo Estimado" (ej. `effort:S`, `effort:M`, `effort:XL`, `effort:XXL`).
        * Crea y aplica etiquetas para el "Epic/Bloque Principal" (ej. `epic:Diseño Web`).
3.  **Añade los Issues a tu GitHub Project:**
    * Ve a tu [Project Board](https://github.com/users/terrerovgh/projects/3).
    * En la columna "Todo", añade los Issues que acabas de crear para el MVP.
4.  **Considera Dividir Tareas Grandes:** Para las tareas marcadas como XL y XXL, es muy recomendable que, una vez creadas como Issues, las desgloses en sub-tareas más pequeñas (puedes usar checklists dentro del Issue principal, o crear Issues separados más pequeños y enlazarlos). Esto te ayudará a gestionar mejor el progreso y tu límite de WIP.
5.  **"Idea Bucket":** Las tareas de ejemplo para el "Idea Bucket" puedes añadirlas directamente a esa columna en tu GitHub Project como "draft issues" o Issues completos cuando quieras empezar a detallarlas más.

Este es tu punto de partida. A medida que avances, moverás las tareas de "Todo" a "In Progress" y luego a "Done", y tu "Idea Bucket" se irá llenando con nuevas ideas o tareas para futuras fases.

¡Espero que esto te ayude a arrancar con tu tablero Kanban!
