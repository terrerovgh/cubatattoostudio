# Agent Execution and Operational Workflows

This document explains the roles of AI agents and the operational workflows that enable seamless interaction and functionality for cubatattoostudio.com. It is designed for autonomous agents and technical teams to coordinate actions and maintain system integrity.

## Agent Roles
- **@CubatattooBuilder:** Full-stack development agent specialized in React, Next.js, and TypeScript. Responsible for implementing features according to the project blueprint.
- **Context Manager:** Ensures all relevant documentation and instructions are loaded and accessible to agents.
- **QA Agent:** Reviews code changes, verifies adherence to standards, and manages approval cycles.

## Operational Workflows
1. **Context Loading:** All documentation is uploaded and referenced before development begins.
2. **Task Assignment:** Features are implemented sequentially, with explicit prompts and review checkpoints.
3. **Component Integration:** Agents follow the AIOp protocol for third-party components, ensuring dependencies and code placement are correct.
4. **Review & Approval:** QA agent or human overseer reviews each change before deployment.
5. **Maintenance:** Agents periodically check for updates to third-party components and reapply the integration protocol as needed.

## Best Practices
- Maintain clear communication and traceability for all agent actions.
- Use version control and changelogs for every workflow step.
- Document all decisions and changes for future reference.
